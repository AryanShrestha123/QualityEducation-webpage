User Profile


The HTML code for the User Profile page is organized into a header, main content, and footer, each of which is composed of various HTML tags that contribute to the overall layout, design, and interactivity of the page.


HTML Techniques

The User Profile page is structured using semantic HTML5 tags for better readability and accessibility. The <header> tag contains the site's title and navigation menu, while the <main> tag houses the primary content, including the user's information and interactive elements. The <footer> tag provides additional links, credits, and contact details.


Key HTML elements include:

  • <nav>: Provides navigation links to essential sections such as Dashboard, Settings, and Profile.
  • <h1>, <h2>: Define headings to create a structured content hierarchy.
  • <p>: Used for paragraphs to maintain readability and structure.
  • <a>: Hyperlinks that allow seamless navigation across different sections and pages.
  • <img>: Displays user profile pictures with descriptive alt attributes for accessibility.
  • <button>: Enables interactive actions such as editing profile details and saving changes.


CSS Techniques

The page utilizes modern CSS styling to ensure a visually appealing and responsive design. Flexbox and Grid are used to align elements efficiently, while media queries make the layout adaptable to different screen sizes. The color scheme and typography align with the overall theme of the website.

Key CSS properties include:

  • A flexbox layout for the overall profile container
  • A card-based design with distinct sections for different types of information
  • A circular profile image using border-radius: 50%
  • Custom styling for the skill bars with progress indicators
  • A responsive contact form with hover effects on the button
  • Consistent spacing and typography throughout the profile

Contact form

  • Uses appropriate input types (text, email) for different data
  • Includes placeholder text for user guidance
  • Uses the required attribute for form validation
  • Provides a submit button using the semantic <button> element


Profile Section

The Pages designed with accessibility in mind. Key features include:

  • Uses semantic <section> elements to organize different types of information
  • Implements skill bars with nested <div> elements and inline width styling to create visual progress indicators
  • Lists experiences and supported UN goals using unordered lists
  • Maintains consistent heading hierarchy with <h3> tags for section titles

SiteMap


The HTML code for the Sitemap page is structured to provide a visual representation of the website's navigation hierarchy using SVG graphics. The page is organized into header, main content with the interactive sitemap, and footer sections.


Key HTML elements include:

  • <svg>: Creates the container for the visual sitemap with specified dimensions
  • <rect>: Generates rectangular boxes representing each page in the sitemap
  • <line>: Creates connecting lines between pages to show the site hierarchy
  • <text>: Adds text labels inside each rectangle to identify the page
  • <a>: Wraps SVG elements to make them clickable and link to corresponding pages


HTML Sitemap Structure

The following sections are included in the sitemap, each representing an essential part of the website's structure:


  • Splash: The landing page of the website that introduces the initiative and its core values.
  • Home: The main homepage with general information about Quality Education and recent updates.
  • Volunteer: A page where users can find information about volunteering opportunities and apply.
  • Table: A section dedicated to showcasing data or information in tabular form related to education statistics.
  • User Profile: A page that allows users to view and update their profile information.
  • Feedback: A form for users to provide feedback or suggestions about the website.
  • Team: A page that introduces the team behind the Quality Education initiative.


Sitemap Structure

The sitemap visually represents the website's structure using SVG elements:

  • Uses nested <a> tags within SVG to create clickable elements
  • Implements a vertical layout with clear visual hierarchy
  • Shows relationships between different page types through connecting lines
  • Positions elements using precise x and y coordinates


CSS Techniques

The page uses CSS to style the SVG elements and create a visually appealing, interactive sitemap. Different colors are used to categorize page types, and hover effects enhance user interaction.


Key CSS properties include:

  • Rounded rectangles using rx: 20px; ry: 20px; for all page boxes
  • Color-coding system with different fill colors for main pages, content pages, and editor pages
  • Hover effect with rect:hover { fill:black; } providing visual feedback
  • Centered layout using display: flex; justify-content: center; for the sitemap container
  • White text on colored backgrounds for readability

Visual Hierarchy

The sitemap employs visual design principles to organize the website structure:

  • Uses color-coding to categorize pages into three types: main navigation pages (red), content pages (purple), and editor pages (blue)
  • Places the splash screen at the top as the entry point to the site
  • Arranges content pages on one side and main navigation pages on the other
  • Shows relationships between pages using connecting lines
  • Maintains consistent sizing and spacing for all page elements


Interactive Elements

The sitemap includes interactive features to enhance usability:

  • All rectangles change color on hover, indicating they are clickable
  • Each rectangle links to its corresponding page through enclosing anchor tags
  • The layout provides a clear visualization of where each page fits in the site hierarchy
  • Alternative text-based navigation is provided in the footer for accessibility


Challenges and Lessons Learned

Creating an SVG-based sitemap presented several technical challenges. Manually positioning elements required careful planning and precise coordinate calculations. Making SVG elements both visually appealing and interactive demanded an understanding of both graphics and hyperlink behavior within SVG contexts. The process taught valuable lessons about structuring complex visual elements, maintaining consistent styling, and ensuring interactive elements work properly across different browsers. Breaking down the sitemap into component parts (rectangles, lines, text) made the implementation more manageable.



Compliance

The sitemap page complies with web standards and Janet regulations through its semantic HTML structure, responsive design approach, and clear information architecture. The SVG elements are scalable by nature, adapting to different screen sizes. The page provides multiple navigation paths through both the visual sitemap and traditional text links. Style and content are properly separated through CSS, following best practices for web development.

Content Web Page


Content Web Page is a web page that provides information about the content of the website. It is designed to be informative, engaging, and visually appealing to visitors. The HTML code for the Content Web Page is structured to include a header, main content, and footer sections, each containing various HTML elements that contribute to the overall layout and design of the page.


Key HTML elements include:

  • <header>: Contains the site's title, logo, and navigation menu for easy access to different sections.
  • <main>: Houses the primary content of the page, including text, images, and interactive elements.
  • <footer>: Provides additional links, credits, and contact information for users to explore further.
  • <h1>, <h2>: Define headings to structure the content hierarchy and improve readability.
  • <p>: Used for paragraphs to present information in a clear and concise manner.
  • <img>: Displays images to complement the text and enhance visual appeal.


Syling using css

The page utilizes CSS to style the content and create an attractive layout that aligns with the overall theme of the website. CSS properties are used to define colors, fonts, spacing, and other visual aspects of the page, ensuring a consistent and visually appealing design.


  • Box-shadow implementation: 0px 4px 6px rgba(0, 0, 0, 0.2)
  • Flexbox layout for page container
  • Image styling: 60% width with border-radius
  • Fixed positioning for "go to top" button


Challenges and Lessoned Learned

One of the challenges faced during the development of this project was ensuring the layout was responsive across different screen sizes. This process taught me the importance of thorough testing on multiple devices for a seamless user experience
